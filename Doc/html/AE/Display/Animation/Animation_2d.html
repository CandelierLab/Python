AE.Display.Animation.Animation_2d

Simple tools for displaying 2D animations

Animation2d

The Animation2d wraps a QGraphicsView and QGraphicsScene as well as all necessary tools for display (scene limits, antialiasing, etc.) Groups of elements can be formed for easier manipulation, and user interaction is possible (drag, click, etc.). It contains a timer triggering the Animation2d.update() method at a regular pace. In subclasses, this allows to change elements' positions or features (color, size, etc.) to create animations.

Items

The items are the elements displayed in the scene (e.g. circles, lines, ...). They derive both from the generic class item and from their corresponding QGraphicsItem. They are incorporaeted in the animation via Animation2d.add().

Simple animation window

The Window class creates a simple window containing the Animation2d.Qview() widget. It manages the QApplication, size on screen, shortcuts and timer trig. If an Animation2d object is created without parent (QWidget or Window), the default Window is automatically created.

 73  """
 74  2D Animation
 75
 76  Base class for two-dimensional animations.
 77
 78  The :py:attr:`Animation2d.Qview` attribute is a ``QGraphicsView`` and can thus
 79  be used directly as a QWidget in any Qt application. For rapid display, the
 80  companion class :class:`Window` allows to easily create a new window for
 81  the animation.
 82
 83  .. note:: Two times are at play in an animation: the *display* time, whose 
 84    increments are approximately the inverse of the :py:attr:`Animation2d.fps`
 85    attribute, and the *animation* time, which is a virtual quantity unrelated 
 86    to the actual time. This way, slow motion or fast-forward animations can
 87    be displayed. The :py:attr:`Animation2d.dt` attibute controls the increment
 88    of animation time between two display updates.
 89  
 90  Attributes:
 91    
 92    item ({:class:`item` *subclass*}): All items in the scene.
 93
 94    boundaries ({'x', 'y', 'width', 'height'}): Limits of the scene.
 95
 96    margin (float): Margin around the scene (pix).
 97
 98    timeHeight (float): Position of the time display relative to the top of the
 99      scene (pix).
100
101    fps (float): Display framerate (1/s). Default is 25.
102
103    t (float): Current animation time (s).
104
105    dt (float): Animation time increment (s) between two updates.
106
107    disp_time (bool): If true, the animation time is overlaid to the animation.
108
109    disp_boundaries (bool): If true, a thin grey rectanle is overlaid to 
110      indicate the boundaries.
111
112    parent (``QWidget`` or :class:`Window`): If not None (default), a simple
113      window containing the animation is created.
114
115    Qscene (``QGraphicsScene``): ``QGraphicsScene`` containing the elements.
116
117    Qview (``QGraphicsView``): ``QGraphicsView`` widget representing the scene.
118
119    timer (``QElapsedTimer``): Timer storing the display time since the 
120      animation start.
121
122    Qtimer (``QTimer``): Timer managing the display updates.
123  """
124

131    Animation constructor
132
133    Defines all the attributes of the animation, especially the ``QGraphicsScene``
134    and ``QGraphicsView`` necessary for rendering.
135
136    Args:
137

139
140      boundaries ([[float,float],[float,float]]): Limits of the scene to display.
141        The first element sets the *x*-limits and the second the *y*-limits. 
142        Default is [[0,1],[0,1]].
143
144      disp_boundaries (bool): If true, a thin grey rectanle is overlaid to 
145        indicate the boundaries.
146
147      disp_time (bool): If true, the animation time is overlaid to the animation.
148
149      dt (float): Animation time increment (s) between two updates. Default: 0.04.
150    """
151
152    super().__init__()
153
154    # --- Size settings
155
156    self.viewHeight = viewHeight if viewHeight is not None else QApplication.desktop().screenGeometry().height()*0.6
157    
158     # --- Scene & view
159
160    # Scene limits
161    self.boundaries = {'x':[0,1], 'y':[0,1], 'width':None, 'height':None}
162    if boundaries is not None:
163      self.boundaries['x'] = list(boundaries[0])
164      self.boundaries['y'] = list(boundaries[1])
165    self.boundaries['width'] = self.boundaries['x'][1]-self.boundaries['x'][0]
166    self.boundaries['height'] = self.boundaries['y'][1]-self.boundaries['y'][0]
167
168    # Scale factor
169    self.factor = self.viewHeight/self.boundaries['height']
170
171    # Scene
172    self.scene = QGraphicsScene()
173    self.view = view(self.scene)
174
175    # Scene boundaries display
176    self.disp_boundaries = disp_boundaries
177    if self.disp_boundaries:
178      self.box = QGraphicsRectItem(0,0,
179        self.factor*self.boundaries['width'],
180        -self.factor*self.boundaries['height'])
181      self.box.setPen(QPen(boundaries_color, 2))
182      self.scene.addItem((self.box))
183
184    # --- Display items
185
186    self.item = {}
187    self.composite = {}
188
189    # --- Display
190
191    # Dark background
192    # self.view.setBackgroundBrush(Qt.black)
193    # pal = self.view.palette()
194    # pal.setColor(QPalette.Window, Qt.black)
195    # self.view.setPalette(pal)
196
197    # Antialiasing
198    self.view.setRenderHints(QPainter.Antialiasing)
199
200    # Stack
201    self.stack = {'vpos': self.boundaries['y'][1], 
202                  'vmin': self.boundaries['y'][0],
203                  'vpadding': 0.02}
204    
205  # ========================================================================
206  def add(self, type, name, **kwargs):
207    """
208    Add an item to the scene.
209
210    args:
211      item (:class:`item` *subclass*): The item to add.
212    """
213
214    # Stack
215    if 'stack' in kwargs:
216      stack = kwargs['stack']
217      del kwargs['stack']
218    else:
219      stack = False
220
221    height = kwargs['height'] if 'height' in kwargs else None
222
223    if height=='fill':
224      height = self.stack['vpos']-self.boundaries['y'][0]
225      kwargs['height'] = height
226      
227    # Add items
228    if issubclass(type, composite):
229
230      # Let composite elements create items
231      self.composite[name] = type(self, name, **kwargs)
232
233    else:
234
235      # Create item
236      self.item[name] = type(self, name, **kwargs)
237
238      # Add item to the scene
239      if self.item[name].parent is None:
240        self.scene.addItem(self.item[name])
241    
242    # --- Stack
243
244    if stack:
245
246      x = 0
247      y = self.stack['vpos']
248
249      if height is None:
250        self.stack['vpos'] -= self.item[name].height()
251      else:
252        self.stack['vpos'] -= height
253        y -= height
254
255      # Set position
256      self.item[name].position = [x, y]
257
258      # Bottom padding
259      self.stack['vpos'] -= self.stack['vpadding']
260
261  # ========================================================================
262  def setPadding(self, padding):
263
264    self.view.padding = padding*self.factor
265
266  # ========================================================================
267  def update(self, t=None):
268    """
269    Update animation state
270    """
271
272    # Repaint
273    self.view.viewport().repaint()
274
275    # Confirm update
276    self.updated.emit()
277

280    """
281    Event reception
282    """
283
284    match event['type']:
285
286      case 'show':

288        for name in self.composite:

291
292      case 'update':
293
294        # Update dispay
295        self.update(event['time'])
296
297      case _:
298        pass
299        # print(event)

2D Animation

Base class for two-dimensional animations.

The Animation2d.Qview attribute is a QGraphicsView and can thus be used directly as a QWidget in any Qt application. For rapid display, the companion class Window allows to easily create a new window for the animation.

Two times are at play in an animation: the display time, whose

increments are approximately the inverse of the Animation2d.fps attribute, and the animation time, which is a virtual quantity unrelated to the actual time. This way, slow motion or fast-forward animations can be displayed. The Animation2d.dt attibute controls the increment of animation time between two display updates.

Attributes:
  • item ({item subclass}): All items in the scene.
  • boundaries ({'x', 'y', 'width', 'height'}): Limits of the scene.
  • margin (float): Margin around the scene (pix).
  • timeHeight (float): Position of the time display relative to the top of the scene (pix).
  • fps (float): Display framerate (1/s). Default is 25.
  • t (float): Current animation time (s).
  • dt (float): Animation time increment (s) between two updates.
  • disp_time (bool): If true, the animation time is overlaid to the animation.
  • disp_boundaries (bool): If true, a thin grey rectanle is overlaid to indicate the boundaries.
  • parent (QWidget or Window): If not None (default), a simple window containing the animation is created.
  • Qscene (QGraphicsScene): QGraphicsScene containing the elements.
  • Qview (QGraphicsView): QGraphicsView widget representing the scene.
  • timer (QElapsedTimer): Timer storing the display time since the animation start.
  • Qtimer (QTimer): Timer managing the display updates.
